# ADR-008: 외부 확장(MCP) 조건부 우선 사용

## 📚 Context

`/hk-extend` 는 Serena(LSP 코드 인텔리전스 MCP) 같은 외부 확장을 opt-in 으로 붙인다. 한편 키트의 핵심 원칙은 "컨텍스트 비용 0 우선 (bash > slash > skill > MCP)"(CLAUDE.md #2) 으로, MCP 를 최후 수단으로 둔다.

이 둘 사이에 공백이 있었다: 확장이 설치돼 있어도 "언제 써야 하는가" 지침이 없어, 에이전트가 설치된 serena 를 두고 raw grep/Read 만 사용하는 일이 실제로 발생했다(설치 ≠ 사용). 확장의 가치는 사용에서 나오므로, 사용 지침이 없으면 상시 컨텍스트 비용만 내고 이득은 못 본다.

## 🎯 Decision

외부 확장은 **조건부 우선 사용** 한다: 확장이 **설치돼 있고** 그 작업이 **그 도구의 강점 영역**(예: LSP 언어의 심볼 정의/참조 추적)일 때, raw grep 다단계보다 확장을 우선 호출한다. 그 외(비-LSP, bash, 단순 단일 grep)에는 원칙 #2 가 그대로 적용되어 확장을 강제하지 않는다. 규칙은 확장 일반으로 기술하고 serena 는 현재 인스턴스로 둔다.

## 📊 Consequences

- **긍정**: 설치된 확장이 강점 영역에서 실제로 사용된다. LSP 언어에서 심볼 작업은 grep+통독보다 호출 수·컨텍스트가 줄어 원칙 #2 의 *정신*(컨텍스트 절약)과 오히려 합치한다.
- **부정**: "강점 영역" 판단을 에이전트에 맡기므로 경계가 흐릴 수 있다 → 규칙에 예시를 명시해 완화.
- **중립**: 원칙 #2 는 폐기가 아니라 *정교화* 된다 — MCP 최후 휴리스틱은 "상시비용이 이득을 못 넘는 경우" 로 좁혀진다.

## 🔀 Alternatives

- **무조건 우선 사용**: 설치되면 항상 확장 우선. 비채택 이유: bash·단순 작업에까지 MCP 호출을 강제해 원칙 #2 와 정면 충돌, 불필요한 상시비용.
- **현행 유지(지침 없음)**: opt-in 만 두고 사용은 에이전트 재량. 비채택 이유: 설치본이 방치되는 실제 문제를 해결 못 함.
