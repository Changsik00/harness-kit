비가역/고위험 변경에 대해 **ship 전** 독립 서브에이전트로 *반증(refute)* 패스를 수행합니다. 목표는 코드 리뷰(커버리지 추가)가 아니라, **"이 변경의 모든 테스트가 통과해도 `spec.md` 에 합의된 *의도*가 깨지는 시나리오"** 를 적극적으로 찾는 것입니다 (#212 검증 2단계).

> 검증 0단계(`check-test-trust.sh`)이 *동어반복·over-mock* 을 싸게 잡는다면, 검증 2단계 는 auto 가 가장 못 잡는 **방향 오류**(코드의 *동작*만 인코딩하고 *의도*는 안 한 경우)를 잡습니다. 검증 2단계 는 LLM 구동이라 hook 이 아니라 본 커맨드로 기동합니다.

## 언제 쓰나 (위험 비례)

라인 수가 아니라 **위험**으로 판단합니다. 다음 신호 중 하나면 ship 전 권장:
- `check-irreversible` 가 감지하는 비가역 행동(force push·외부 발행·history rewrite 등)을 동반/유발하는 변경
- `check-scope` 위험 신호 또는 결제·인증·삭제·마이그레이션 등 blast-radius 가 넓은 변경
- auto 모드에서 사람 검토 없이 진행된 변경 (phase-ship 전 일괄)

저위험(문서·로그 포매터·국소 리팩터)에는 호출하지 않습니다 — ceremony 비용 통제(ADR-002).

## 1. 대상 확인

```bash
bash .harness-kit/bin/sdd status --json
```

활성 spec 의 `specs/<spec-dir>/spec.md` 와 현재 변경(`git diff` 또는 PR diff)을 대상으로 합니다. 활성 spec 이 없으면 사용자에게 보고하고 멈춥니다.

## 2. Opus 서브에이전트로 반증 수행

Agent tool (`subagent_type: general-purpose`, `model: "opus"`) 로 **별도 컨텍스트**에서 수행합니다. 디렉터는 구현 추론을 전달하지 않습니다 — 독립성의 원천은 "다른 모델"이 아니라 **구현 멘탈 상태 미공유**(#212).

서브에이전트에게 전달할 프롬프트:

> 당신은 독립적인 **적대적 검증자**입니다. 목표는 커버리지 추가가 아니라 **반증(refute)** 입니다.
>
> **입력 자료**:
> - 합의된 *의도*: `specs/<spec-dir>/spec.md` (배경·요구사항·Out of Scope·DoD). 이것이 진실의 기준입니다.
> - 관찰된 *변경*: `git diff <base>...<head>` 의 코드·테스트.
> - **금지**: 구현의 커밋 메시지·주석·walkthrough 의 자기 정당화를 신뢰하지 마세요. 같은 오해를 물려받습니다. 의도와 관찰된 동작만 보세요.
>
> **과제**: "이 변경의 **모든 테스트가 통과해도** `spec.md` 의 의도가 깨지는 시나리오"를 적극적으로 찾으세요. 다음을 의심합니다:
> - **동어반복**: 테스트가 구현과 같은 오해를 공유해 틀려도 초록불인가? 테스트가 *의도*가 아니라 *현재 코드의 동작*을 그대로 박제했는가?
> - **가짜 red**: 올바른 이유로 먼저 실패하는 red 단계가 있었는가? 구현을 망가뜨리면 어떤 테스트가 실제로 빨개지는가?
> - **방향 오류**: 요구사항/Out of Scope 와 어긋나지만 테스트는 통과하는 경로가 있는가?
>
> **출력**:
> 1. 의도-위반 가능 시나리오 목록 — 각: (트리거 조건 / 왜 현재 테스트가 못 잡는지 / 어떤 의도 항목을 깨는지)
> 2. 심각도와 함께 **Go / No-Go** 권고. 불확실하면 No-Go 쪽으로 기웁니다.

## 3. 종합 (디렉터)

서브에이전트 결과를 검토합니다:
- 실재하는 의도-위반이면 → spec/구현 보정 또는 ship 보류(STOP), 사용자에 보고.
- 반증 실패(견고)면 → 그대로 ship 진행. 결과 요지를 `walkthrough.md` 발견 사항에 1줄 남깁니다.

> 본 커맨드는 검증 2단계 **골격**입니다 — 트리거 신호·절차·디스패치를 규정합니다. 완전 자동(고위험 감지 시 자동 호출)은 운영 데이터 축적 후 검토합니다 (spec-25-02 Out of Scope).
