  - **합의**: 용어를 "검증 N단계"로(번호↑=강도·비용↑), `/hk-auto` 추가. 둘을 한 spec-x로.

## 🔍 발견 사항

- **순서 있는 치환이 필요했다**: "비용 사다리의 칸N"을 단순 치환하면 "검증 단계의 검증 N단계" 중복이 생긴다. (1)복합 "사다리의 칸N" → (2)"사다리" → (3)"칸N" 순으로 처리. 그래도 "비용 사다리 칸2"(조사 없는 병렬) 형태가 "위험 비례 검증 단계 검증 2단계" 중복을 남겨 2곳 수동 보정.
- **"검증 단계"가 자명한 이유**: 번호가 곧 강도·비용 레벨(0=싼 정적, 2=비싼 적대적)이라, 비유(사다리/칸) 없이 숫자만으로 읽힌다. 사용자 직감이 옳았다.
- **역사 기록은 안 건드림**: phase-25.md(완료)·머지된 walkthrough의 "칸N"은 grep 으로 추적 가능한 당시 명칭이라 보존. 운영 정합만 확보.
