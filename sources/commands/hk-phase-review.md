Phase ship 전에 실행하여 Phase 전체를 회고합니다.

## 1. Phase 정보 수집

```bash
./.harness-kit/bin/sdd phase show
```

- `backlog/phase-{N}.md` 에서 목표, 성공 기준, spec 목록을 읽습니다.
- 각 spec 의 `walkthrough.md` 를 확인하여 실제 구현 내용을 파악합니다.

## 2. 회고 항목

아래 항목을 순서대로 평가하고 한국어로 보고합니다.

### 2-1. 목표 달성도

- Phase 목표 (phase.md "🎯 배경 및 목표" 섹션) 를 다시 읽고, 각 성공 기준이 충족되었는지 ✅/❌ 로 판정합니다.
- 초기 계획 대비 추가/삭제/변경된 spec 이 있는지 확인합니다.

### 2-2. 테스트 품질

- 각 spec 의 테스트 결과를 종합합니다 (walkthrough.md 의 검증 결과 섹션).
- 테스트가 "의미 있는 검증" 인지, 단순 통과용인지 평가합니다.
- 통합 테스트 커버리지가 성공 기준을 충분히 다루는지 확인합니다.

### 2-3. 거버넌스 변경 추적

- Phase 기간 중 `constitution.md`, `agent.md`, `align.md` 에 변경이 있었는지 확인합니다.
- 변경이 있었다면: 왜 변경했는지, 초기 규칙이 부적절했는지, 진화인지 판단합니다.

### 2-4. 프로세스 교훈

- Phase 진행 중 마찰이 있었던 지점을 회고합니다.
- 잘 된 점 (Keep), 개선할 점 (Improve), 시도할 점 (Try) 을 KIT 형식으로 정리합니다.

### 2-5. 에이전트 소견

- 에이전트 관점에서 본 Phase 의 강점, 약점, 다음 Phase 에 반영할 제안을 자유롭게 기술합니다.
